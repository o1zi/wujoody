# رِواق — دليل المشروع الشامل

منصّة سعودية (SaaS) تتيح لأي **مكتب هندسي** (وأي نشاط لاحقاً) إنشاء **موقع احترافي بنطاق فرعي خاص** خلال دقائق، مع لوحة تحكم متكاملة، واشتراك مدفوع عبر **سلة** بتجديد شهري تلقائي.

- **الاسم التجاري:** رِواق (RIWAQ)
- **المستودع:** https://github.com/o1zi/wujoody
- **النشر:** Vercel — https://wujoody.vercel.app

---

## 1) نظرة عامة

كل مكتب يحصل على موقع ديناميكي واحد يُبنى من قالب سينمائي موحّد، ويُخصّصه المكتب بالكامل من لوحة تحكمه (المحتوى، الأقسام، الألوان، الخطوط، الخلفية). الموقع يظهر على:
- **نطاق فرعي**: `office.example.com` (عبر إعادة كتابة في الـ proxy).
- أو **مسار** على نطاق vercel.app: `wujoody.vercel.app/s/<slug>` (احتياطي عندما لا تتوفر نطاقات فرعية).
- أو **نطاق المكتب الخاص** (.com/.sa) بعد ربطه.

تُفعّل المواقع تلقائياً بعد الدفع، وتُغلق تلقائياً عند انتهاء/إلغاء الاشتراك (مع بقاء المحتوى محفوظاً).

## 2) التقنية (Tech Stack)

| المكوّن | التقنية |
|--------|---------|
| الإطار | Next.js (App Router, TypeScript, React 19, Tailwind v4) |
| قاعدة البيانات/الهوية | Supabase (Postgres + RLS + Auth + Storage) |
| المدفوعات | سلة (Salla) — روابط دفع + Webhook + تجديد تلقائي |
| الإيميل | Resend (اختياري) |
| الإشعارات الفورية | Telegram Bot |
| الذكاء الاصطناعي | Anthropic (Claude Haiku) لتوليد المحتوى |
| الاستضافة | Vercel (+ Vercel Cron) |

ملاحظة: هذه نسخة Next.js حديثة قد تختلف عن المعتاد — راجع `node_modules/next/dist/docs/` قبل أي تعديل بنيوي. الـ middleware مُسمّى `proxy.ts` بدالة `proxy`.

## 3) المعمارية (Architecture)

- **متعدّد المستأجرين (Multi-tenant):** موقع واحد ديناميكي يقرأ محتوى كل مكتب من قاعدة البيانات.
- **توجيه النطاقات:** `src/proxy.ts` يكتشف النطاق الفرعي ويعيد الكتابة إلى `/s/[subdomain]/...`. المسارات المحجوزة (`/site-template`, `/backgrounds`, `/_next`, `/api`, `/auth`) لا يُعاد كتابتها.
- **مصدر الحقيقة للبيانات:** جداول Supabase. الكود يحوي قيماً احتياطية (FALLBACK) تُستخدم فقط عند غياب الجداول.
- **الصلاحيات:** RLS على مستوى الصف؛ عميل admin (service_role) للعمليات الخادمية فقط (`src/lib/supabase/admin.ts` بـ `import "server-only"`).

### الأدوار
1. **السوبر أدمن** (مالك المنصة): يدير كل المكاتب، الباقات وصلاحياتها، صفحة الهبوط، الدعم، النطاقات.
2. **مدير المكتب** (المشترك): يدير موقعه، محتواه، عملاءه، اشتراكه، مدوّنته، نطاقه، إشعاراته.
3. **زائر الموقع** (عميل المكتب): يتصفّح موقع المكتب ويتواصل/يحجز.

## 4) الباقات والصلاحيات

ثلاث باقات (الأسعار والمزايا قابلة للتعديل من لوحة السوبر أدمن، ومصدرها جدول `plans`):

| الباقة | السعر/شهر | أبرز ما تشمله |
|--------|-----------|----------------|
| **الأساسية** | 249 ر.س | موقع بنطاق فرعي، أنماط جاهزة، محرّر متكامل، نموذج تواصل + إشعار بريد/تيليجرام، SEO، خلفية لون ثابت |
| **الاحترافية** | 499 ر.س | + خلفيات فيديو/حركة، واتساب فوري، حجز استشارة، حاسبة تكلفة، صفحات مشاريع تفصيلية، شارات اعتماد، بروفايل PDF، مدوّنة، ربط نطاق خاص |
| **بريميوم** | 899 ر.س | + رفع خلفية خاصة، CRM وتصدير، محتوى بالذكاء الاصطناعي، تقرير أداء شهري |

### نظام الصلاحيات (`PlanCaps`)
كل باقة تحمل في `caps` (jsonb): أعلام مزايا + **قائمة الأقسام المسموحة** `sections`. التحكم بالأقسام لكل باقة يتم من **السوبر أدمن ← إدارة الباقات** (خانات اختيار لكل قسم). الأعلام `badges/booking/blog` تُشتق تلقائياً من قائمة `sections` (مصدر واحد للحقيقة).

- الأقسام: `about, services, stats, process, projects, team, testimonials, credentials, faq, booking, calculator, blog, contact`.
- مزايا أخرى: `solidOnly, presets, presetLimit, upload, whatsapp, projectDetails, profilePdf, customDomain, crm, aiContent, aiMonthlyLimit, monthlyReport`.

القسم يظهر على الموقع فقط إذا: **الباقة تسمح به** *و* **المكتب لم يُخفه**. الأقسام غير المتاحة بالباقة تظهر مقفلة (🔒) في محرّر المكتب.

## 5) المزايا بالتفصيل

### 5.1 موقع المكتب (للزائر)
قالب سينمائي واحد (`SiteView`) بأقسام قابلة للإظهار/الإخفاء والتخصيص:
- **الواجهة**: عنوان، وصف، أرقام بارزة، خلفية (لون ثابت / فيديو / إطارات متحركة مع التمرير).
- **من نحن، الخدمات، الأرقام، المنهجية، الفريق، آراء العملاء**.
- **المشاريع**: معرض، ومع الباقة الاحترافية **دراسة حالة** لكل مشروع (وصف مطوّل + معلومات + صور إضافية) تُفتح في نافذة زجاجية.
- **الاعتمادات والثقة**: ترخيص/هيئة/تصنيف + شعارات عملاء.
- **الأسئلة الشائعة**: أكورديون بدون جافاسكربت.
- **حجز استشارة**: نموذج بتاريخ/وقت مفضّل → يصل كـ lead بشارة «حجز».
- **حاسبة تقدير التكلفة**: الزائر يُدخل المساحة + نوع الخدمة + مستوى التشطيب → تقدير نطاقي فوري + زر «اطلب عرضاً دقيقاً».
- **التواصل**: نموذج + خريطة جوجل + أزرار تواصل عائمة (واتساب/تيك توك/سناب/إنستقرام/لينكدإن) + **زر واتساب فوري** يفتح محادثة بالمكتب ببيانات الزائر مُعبّأة.
- **المدوّنة**: صفحات `/blog` و`/blog/[slug]` بنفس هوية الموقع + SEO و Article JSON-LD.

### 5.2 مظهر الموقع (الثيم)
- **معرض أنماط جاهزة** (6 أنماط) يطبّق بنقرة: الخط + اللون + شكل البطاقات + الخلفية. + زر **«استعادة النمط الأساسي»**.
- **الخط**: 12 خطاً عربياً (Google Fonts) — `src/lib/site-fonts.ts`.
- **شكل البطاقات**: زجاجي / صلب / محدّد، واستدارة (حادّة/ناعمة/دائرية)، و**لون بطاقات مع الحفاظ على الشفافية**.
- **لون تمييز مخصّص** (يتجاوز الألوان الجاهزة).
- يُطبّق عبر متغيّرات CSS على غلاف الموقع (`src/lib/site-theme.ts`) ويُستخدم أيضاً في صفحات المدوّنة.

### 5.3 لوحة مدير المكتب (`/dashboard`)
- **نظرة عامة** + لافتة **معالج الإعداد** حتى يكتمل.
- **معالج الإعداد (Onboarding)**: 4 خطوات (هوية+شعار+لون، نصوص مع خيار توليد AI، تواصل، خلفية) ثم التوجيه للاشتراك. يعلّم المكتب `onboarded`.
- **محرّر الموقع**: كل المحتوى + مظهر الموقع + الأقسام الظاهرة + رمز تحقق Google + زر **«تحميل الملف التعريفي PDF»**.
- **الرسائل (Leads)**: رسائل التواصل والحجوزات (بشارة «حجز»). للبريميوم **CRM**: حالة (جديدة/تم التواصل/مكتسب/غير مهتم/مؤرشفة) + ملاحظة + **تصدير CSV**.
- **الإشعارات**: ربط تيليجرام لاستلام العملاء فوراً.
- **التحليلات** (بريميوم): زيارات/نقرات/رسائل + زر **«أرسل تقريري الآن»**.
- **المدوّنة** (احترافية+): إنشاء/تعديل/حذف، مسودة/نشر، رفع غلاف.
- **النطاق الخاص** (احترافية+): إدخال النطاق + تعليمات DNS + تحقّق فعلي.
- **الاشتراك**: عرض الباقات والدفع + تعليمات الإلغاء.
- **الدعم الفني**: محادثة مع السوبر أدمن + أسئلة شائعة جاهزة.

### 5.4 لوحة السوبر أدمن (`/super-admin`)
- إدارة كل المكاتب (الحالة، الباقة، النطاق الفرعي، الحذف، تعديل أي موقع).
- **إدارة الباقات**: السعر، المزايا، رابط الدفع، رقم منتج سلة، الحالة، و**الأقسام المتاحة لكل باقة**.
- **محرّر صفحة الهبوط**، والدعم الفني، وإحصاءات شاملة (زيارات/نقرات/رسائل).
- إيميل السوبر أدمن مقصور عبر `SUPER_ADMIN_EMAILS`.

### 5.5 المحتوى بالذكاء الاصطناعي (بريميوم)
زر «اكتب لي» في المحرّر/المعالج → `/api/ai/generate` (Anthropic Haiku) يولّد «من نحن» و6 خدمات. محدود بـ **10 توليدات/شهر لكل مكتب** (`ai_usage` + `caps.aiMonthlyLimit`)، ويعمل عند ضبط `ANTHROPIC_API_KEY`.

## 6) تكامل سلة (المدفوعات والتجديد)

- **الدفع**: لكل باقة رابط دفع سلة + رقم منتج، يُضافان من محرّر الباقات. صفحة الاشتراك تضيف `?office=<id>` لرابط الدفع لربط الدفعة بالمكتب.
- **الويبهوك** `POST /api/salla/webhook`: يتحقق من التوقيع (HMAC في `x-salla-signature`) أو التوكن (`Authorization`) مقابل `SALLA_WEBHOOK_SECRET`. **idempotent** عبر تخزين كل تسليم في `salla_events` (sha256). عند دفعة ناجحة:
  1. يربط المكتب: عبر **`?office=`** أولاً، ثم **مطابقة الجوال**، ثم **البريد**.
  2. يفعّل/يمدّد الاشتراك 30 يوماً ويضبط حالة المكتب `active`.
  3. يرسل بريد تأكيد (إن توفّر Resend).
- **التجديد التلقائي**: سلة تدعمه (فعّل «السماح بالاشتراك» للمنتج + خطة شهرية). كل دفعة تجديد = حدث دفع → نمدّد 30 يوماً. التجديد التلقائي **بلا `?office=`** فيعتمد على مطابقة الجوال/البريد.
- **الإغلاق/إعادة الفتح**: عند توقّف التجديد، تنتهي المدة فيُغلق الموقع (الكرون + الفحص عند العرض)، **والمحتوى يبقى محفوظاً**. أي دفعة لاحقة تعيد التفعيل تلقائياً.
- **الإلغاء**: يتم من حساب العميل في متجر سلة (تعليمات في صفحة الاشتراك).
- **التشخيص**: `SALLA_WEBHOOK_DEBUG=1` يخزّن أول payload خام (حتى لو فشل التوقيع) في `salla_events` لمعرفة الصيغة الفعلية.

## 7) تكامل تيليجرام (إشعارات فورية)

- بوت واحد للمنصة (`@riwaqbot_bot`). المكتب يربط حسابه من **الإشعارات ← «اربط تيليجرام»** (deep-link بـ `/start <token>`).
- `POST /api/telegram/webhook` يربط `telegram_chat_id` بالمكتب عبر `telegram_link_token`.
- عند كل عميل/حجز جديد → إشعار فوري للمكتب. والتقرير الشهري يُرسل عبر تيليجرام (وإلا بالبريد).
- إعداد لمرة واحدة: إنشاء البوت في BotFather، ضبط `TELEGRAM_BOT_TOKEN` و`TELEGRAM_BOT_USERNAME`، ثم `setWebhook` إلى `/api/telegram/webhook`.

## 8) SEO

- **لكل مكتب**: `/robots.txt` (يسمح للمواقع النشطة) + `/sitemap.xml` (الرئيسية + المدوّنة + المقالات).
- **رمز تحقق Google Search Console** (`content.seo.googleVerification`) يُحقن كوسم meta.
- **للمنصة**: `robots.ts` + `sitemap.ts` بالجذر.
- بيانات منظّمة (JSON-LD) لكل موقع ومقال، وميتا OG/canonical.

## 9) متغيّرات البيئة (Vercel)

| المتغيّر | الغرض |
|---------|-------|
| `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` | عميل Supabase العام |
| `SUPABASE_SERVICE_ROLE_KEY` | عميل admin خادمي فقط — **سرّي، لا يُكشف للمتصفح** |
| `NEXT_PUBLIC_APP_URL` | رابط المنصة (مثل https://wujoody.vercel.app) |
| `NEXT_PUBLIC_ROOT_DOMAIN` | النطاق الجذر لاكتشاف النطاقات الفرعية |
| `SUPER_ADMIN_EMAILS` | إيميلات السوبر أدمن (مفصولة بفواصل) |
| `SALLA_WEBHOOK_SECRET` | سر التحقق من ويبهوك سلة |
| `SALLA_WEBHOOK_DEBUG` | `1` لالتقاط أول payload (مؤقت) |
| `CRON_SECRET` | حماية مسارات الكرون |
| `RESEND_API_KEY` / `EMAIL_FROM` | إرسال البريد (اختياري) |
| `ANTHROPIC_API_KEY` | توليد المحتوى بالذكاء الاصطناعي |
| `TELEGRAM_BOT_TOKEN` / `TELEGRAM_BOT_USERNAME` | بوت تيليجرام |
| `TELEGRAM_WEBHOOK_SECRET` | (اختياري) تحقّق ويبهوك تيليجرام |

> أمان: أي مفتاح سرّي يجب ألا يُلصق في الكود ولا يُرفع على GitHub. `.env.local` مُتجاهَل. إذا انكشف مفتاح، دوّره فوراً.

## 10) قاعدة البيانات

- **`supabase/schema.sql`**: الجداول الأساسية (offices, profiles, subscriptions, site_content, salla_events, leads) + RLS + الدوال + trigger إنشاء المستخدم.
- **`supabase/setup-all.sql`**: ملف **تشغيل واحد** (idempotent) يضيف بقية الجداول/الأعمدة: `plans` (مع البذور والصلاحيات)، `app_settings`، `support_messages`، `site_events`، `posts` (المدوّنة)، `ai_usage`، أعمدة (`profiles.phone`, `leads.kind/note`, `offices.custom_domain/domain_status/onboarded/telegram_*`) + سياسات التخزين.

**مهم:** أي ميزة جديدة تتطلب إعادة تشغيل `setup-all.sql` في Supabase SQL Editor.

## 11) النشر والتشغيل

1. شغّل `supabase/schema.sql` ثم `supabase/setup-all.sql` في Supabase.
2. اضبط متغيّرات البيئة في Vercel وانشر.
3. سلة: وثّق المتجر، أنشئ 3 منتجات (نوع «خدمة حسب الطلب») بأسعار 249/499/899، فعّل «السماح بالاشتراك» + خطة شهرية، والصق روابط الدفع وأرقام المنتجات في محرّر الباقات. اضبط ويبهوك سلة → `/api/salla/webhook`.
4. تيليجرام: أنشئ البوت واضبط التوكن واسم المستخدم و`setWebhook`.
5. Search Console: كل مكتب يضيف رمز التحقق ويرسل `…/sitemap.xml`.

### مسارات الكرون (`vercel.json`)
- `/api/cron/expire` يومياً (إنهاء/تعليق المنتهية + تذكير قبل 3 أيام).
- `/api/cron/monthly-report` شهرياً (تقرير الأداء عبر تيليجرام/البريد).

## 13) ملاحظات مهمة

- **الباقات تُقرأ من القاعدة**: تعديل الكود الاحتياطي لا يظهر إلا إذا كان جدول `plans` فارغاً؛ للتعديل الحي استخدم محرّر الباقات أو أعد تشغيل `setup-all.sql`.
- **مرونة الأعمدة الجديدة**: مسارات `leads`/الإشعارات تتحمّل غياب الأعمدة قبل تشغيل `setup-all.sql` (best-effort) فلا تتعطّل.
- **التحقق قبل النشر**: `npx tsc --noEmit` ثم `npm run build`.
